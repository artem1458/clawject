Custom Scopes

Out of the box, Clawject provides 3 scopes: singleton, transient and fresh, but sometimes you may need to define your own scopes.

For example, if you are developing an http server application, you may want to provide a separate instance of a particular bean or set of beans for each request. Clawject provides a mechanism for creating custom scopes for scenarios such as this.

Creating a CustomScope Classâ€‹

To create a custom scope, you should implement the CustomScope interface.

In the following steps, we will implement http-request scope that is using AsyncLocalStorage to assign and retrieve httpRequestId.

Creating the server codeâ€‹

First of all - let's define HttpExecutionContext class that will assign unique id to each http request, allows us to retrieve currentRequestId, and notify subscriber when http request is finished:

import http from 'node:http';
import { AsyncLocalStorage } from 'node:async_hooks';

export class HttpExecutionContext {
  private static idSeq = 0;
  private static asyncLocalStorage = new AsyncLocalStorage<number>();
  private static declare requestEndSubscriber: (requestId: number) => void;

  static run(fn: () => void) {
    this.asyncLocalStorage.run(this.idSeq++, fn);
  }

  static getCurrentRequestId(): number {
    //For simplicity - let's assume that AsyncLocalStorage always returns a value
    return this.asyncLocalStorage.getStore();
  }

  static onRequestEnd(): void {
    this.requestEndSubscriber(this.getCurrentRequestId());
  }

  static subscribeOnRequestEnd(subscriber: (requestId: number) => void): void {
    this.requestEndSubscriber = subscriber;
  }
}

http.createServer((req, res) => {
  HttpExecutionContext.run(() => {
    setTimeout(() => {
      res.end();
      HttpExecutionContext.onRequestEnd();
    }, 100);
  });
}).listen(8080);

Managing the Scoped Objects and Callbacksâ€‹

One of the first things to consider when implementing a CustomScope class is how you will store and manage the scoped objects and destruction callbacks. This could be done using a Map or a dedicated class, for example.

Now let's define our custom scope class:

class RequestScope implements CustomScope {
  // requestId to Map of objectName to object
  private scopedObjects = new Map<number, Map<string, any>>();
  // requestId to Map of objectName to object destruction callback
  private destructionCallbacks = new Map<number, Map<string, () => void>>();

  /* ... */
}

Retrieving an Object from Scopeâ€‹

To retrieve an object by name from our scope, let's implement the get method. Note that if the named object does not exist in the scope, this method must create and return a new object.

In our implementation, we check to see if the named object is in our map under current requestId. If it is, we return it, and if not, we use the ObjectFactory to create a new object, add it to our map, and return it:

class RequestScope implements CustomScope {
  /* ... */

  get(name: string, objectFactory: ObjectFactory): ObjectFactoryResult {
    const currentRequestId = HttpExecutionContext.getCurrentRequestId();

    const scopedObjects = this.scopedObjects.get(currentRequestId) ?? new Map<string, any>();
    this.scopedObjects.set(currentRequestId, scopedObjects);

    const scopedObject = scopedObjects.get(name) ?? objectFactory.getObject();
    scopedObjects.set(name, scopedObject);

    return scopedObject;
  }
}

Registering a Destruction Callbackâ€‹

We must also implement the registerDestructionCallback method. This method provides a callback that is to be executed when the named object is destroyed or if the scope itself is destroyed by the application:

class RequestScope implements CustomScope {
  /* ... */

  registerDestructionCallback(name: string, callback: () => void): void {
    const currentRequestId = HttpExecutionContext.getCurrentRequestId();
    const scopedDestructionCallbacks = this.destructionCallbacks.get(currentRequestId) ?? new Map<string, () => void>();
    this.destructionCallbacks.set(currentRequestId, scopedDestructionCallbacks);

    scopedDestructionCallbacks.set(name, callback);
  }
}

Removing an Object from the Scopeâ€‹

Next, let's implement the remove method, which removes the named object from the scope and also removes its registered destruction callback, returning the removed object.

Note that it is the caller's (clawject) responsibility to actually execute the callback and destroy the removed object.

class RequestScope implements CustomScope {
  /* ... */

  remove(name: string): ObjectFactoryResult | null {
    const currentRequestId = HttpExecutionContext.getCurrentRequestId();

    const scopedObject = this.scopedObjects.get(currentRequestId)?.get(name) ?? null;

    this.scopedObjects.get(currentRequestId)?.delete(name);
    this.destructionCallbacks.get(currentRequestId)?.delete(name);

    return scopedObject;
  }
}

Destroying the Scopeâ€‹

Finally, let's implement the destroy method, that will be called when http-request will be finished and which destroys the scope and all of its objects:

class RequestScope implements CustomScope {
  constructor() {
    HttpExecutionContext.subscribeOnRequestEnd(this.destroy)
  }

  private destroy = (requestId: number): void => {
    this.scopedObjects.delete(requestId);
    this.destructionCallbacks.get(requestId)?.forEach(callback => callback());
  }

  /* ... */
}

Registering the Scopeâ€‹

To make the Clawject container aware of your new scope, you need to register it through the registerScope method on a ContainerManager instance.

Let's take a look at this method's definition:

registerScope(scopeName: string, scope: CustomScope): void

The first parameter, scopeName, is used to identify/specify a scope by its unique name. The second parameter, scope, is an actual instance of the CustomScope implementation that you wish to register and use.

Let's register our RequestScope:

ScopeRegister.registerScope('http-request', new RequestScope());

Using the Custom Scopeâ€‹

Now that we have registered our custom scope, we can apply it to any of our beans just as we would with any other bean that uses a scope other than singleton (the default scope) â€” by using the @Scope decorator and specifying our custom scope by name.

ScopedBean class will assign a random value to its scopedProperty property that will be different for each request but identical in bounds of one request.

How it works?â€‹

When ScopedBean will be requested from the container - proxy will be injected instead of the actual bean instance, this proxy will ask RequestScope to get the bean instance, RequestScope will return or create bean that is associated with current request id. Because proxies are injected - no needs to rebuild the whole dependency tree, and other beans that is using ScopedBean will have own scope.

export class ScopedBean {
  scopedProperty: number

  constructor() {
    this.scopedProperty = Math.random();
  }
}

export class NonScopedBean {
  constructor(private scopedBean: ScopedBean) {}

  invoke(): void {
    console.log(`scopedProperty of ScopedBean: ${this.scopedBean.scopedProperty}`);
  }
}

export class ApplicationContext extends CatContext {
  @Scope('http-request') scopedBean = Bean(ScopedBean);

  nonScopedBean = Bean(NonScopedBean);
}

Now let's put everything togetherâ€‹

index.ts
import http from 'node:http';
import { ContainerManager } from '@clawject/di';
import { HttpExecutionContext } from './HttpExecutionContext';
import { ApplicationContext } from './ApplicationContext';

const {
  nonScopedBean,
} = ContainerManager.init(ApplicationContext).getBeans();

http.createServer((req, res) => {
  HttpExecutionContext.run(() => {
    setTimeout(() => {
      nonScopedBean.invoke();
      res.end();
      HttpExecutionContext.onRequestEnd();
    }, 100);
  });
}).listen(8080);

RequestScope.ts
import { CustomScope, ObjectFactory, ObjectFactoryResult } from '@clawject/di';
import { HttpExecutionContext } from './HttpExecutionContext';

export class RequestScope implements CustomScope {
  constructor() {
    HttpExecutionContext.subscribeOnRequestEnd(this.destroy)
  }

  // requestId to Map of objectName to object
  private scopedObjects = new Map<number, Map<string, any>>();
  // requestId to Map of objectName to object destruction callback
  private destructionCallbacks = new Map<number, Map<string, () => void>>();

  get(name: string, objectFactory: ObjectFactory): ObjectFactoryResult {
    const currentRequestId = HttpExecutionContext.getCurrentRequestId();

    const scopedObjects = this.scopedObjects.get(currentRequestId) ?? new Map<string, any>();
    this.scopedObjects.set(currentRequestId, scopedObjects);

    const scopedObject = scopedObjects.get(name) ?? objectFactory.getObject();
    scopedObjects.set(name, scopedObject);

    return scopedObject;
  }

  registerDestructionCallback(name: string, callback: () => void): void {
    const currentRequestId = HttpExecutionContext.getCurrentRequestId();
    const scopedDestructionCallbacks = this.destructionCallbacks.get(currentRequestId) ?? new Map<string, () => void>();
    this.destructionCallbacks.set(currentRequestId, scopedDestructionCallbacks);

    scopedDestructionCallbacks.set(name, callback);
  }

  remove(name: string): ObjectFactoryResult | null {
    const currentRequestId = HttpExecutionContext.getCurrentRequestId();

    const scopedObject = this.scopedObjects.get(currentRequestId)?.get(name) ?? null;

    this.scopedObjects.get(currentRequestId)?.delete(name);
    this.destructionCallbacks.get(currentRequestId)?.delete(name);

    return scopedObject;
  }

  private destroy = (requestId: number): void => {
    this.scopedObjects.delete(requestId);
    this.destructionCallbacks.get(requestId)?.forEach(callback => callback());
  }
}
